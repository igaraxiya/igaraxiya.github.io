Git学习笔记

2021-03-27
工具-Git, code

Git 是一个开源的分布式版本控制系统，用于敏捷高效地处理任何或小或大的项目。

Git 是 Linus Torvalds 为了帮助管理 Linux 内核开发而开发的一个开放源码的版本控制软件。

Git 与常用的版本控制工具 CVS, Subversion 等不同，它采用了分布式版本库的方式，不必服务器端软件支持。

Git安装与配置

  • 在Ubuntu上使用sudo apt-get install git安装

  • 在windows上访问Git官网下载安装

安装完成后还需要进行配置，输入：

git config --global user.name "Your Name"
git config --global user.email "email@example.com"

其中--global表示整台机器的Git都会使用这个配置，如果没有该参数则只会应用与当前仓库

每个仓库的配置文件都放在了.git/config文件中。

可以使用：

git config --list

显示当前的Git配置信息。

Git 基本操作

创建仓库

初始化一个新仓库：

git init <repo>

Git会在repo指定的目录下新建一个Git仓库，为指定则会在当前目录新建

从现有仓库中拷贝：

git clone <url> <directory>

Git会从url指定的仓库从克隆到repo中，repo未指定则克隆到在当前目录

示例如下：

git clone git://github.com/username/directory.git

常用文件操作

命令（文件） 使用示范 说明
git add git add ./git add file 添加文件到仓库
git commit git commit -m "xxx" 提交暂存区到本地仓库
git statues git status 查看仓库当前的状态，显示有变更的文件
git diff git diff file 比较暂存区和工作区文件的差异
git reset 回退版本
git rm 删除文件

将指定的工作区文件或文件夹添加至暂存区：

git add <file1> <flie2> ...

或直接添加当前工作区的所有文件到暂存区：

git add .  #提交被修改(modified)和新文件(new)文件，不包括被删除(deleted)文件
git add -u #提交被修改(modified)和被删除(deleted)文件，不包括新文件(new)
git add -A #提交所有变化（git add --all 的缩写）

注意Git不会在成功执行add后提示任何信息，没有消息就是好消息。

将暂存区文件提交至本地仓库，使用：

git commit -m "description"

其中-m后面的description表示对本次提交的说明，当然你可以不写，等回退的时候再后悔。

执行git commit后Git会告诉你文件的变动信息



修改已经上传至GIthub的文件

git rm -r --cached .idea  #--cached不会把本地的 .idea 删除

git push -u origin master

远程仓库

命令（远程仓库） 使用示范 说明
git push git push origin master 提交至远程仓库
git pull
git clone git clone url 从远程仓库克隆

以Github为例：

ssh-keygen -t rsa -C "youremail@example.com"

在电脑上添加SSH密钥，然后将公钥添加到GIthub：

cd ~/.ssh
cat id_rsa.pub

将显示的内容复制到Github上。

在Git上新建一个仓库，使用：

git remote add origin git@github.com:username/repository.git

将本地仓库与一个远程仓库关联。然后使用：

git push -u origin master

将本地的master分支提交到远程，加入-u参数还会把本地的master分支和远程的master分支关联起来

git push的不同参数如下：

git push origin master：refs/for/master
 #git push <远程主机名> <本地分支名> <远程分支名>

git push origin master
 #如果远程分支被省略，如上则表示将本地分支推送到与之存在追踪关系的远程分支（通常两者同名）
 #如果该远程分支不存在，则会被新建

git push origin ：refs/for/master
 #如果省略本地分支名，则表示删除指定的远程分支，因为这等同于推送一个空的本地分支到远程分支
 #等同于 git push origin –delete master

git push origin
 #如果当前分支与远程分支存在追踪关系，则本地分支和远程分支都可以省略，将当前分支推送到origin主机的对应分支

git push
 #如果当前分支只有一个远程分支，那么主机名都可以省略，形如 git push，可以使用git branch -r ，查看远程的分支名

分支管理

主分支变main了

命令（分支） 使用示范 说明
git branch git branch wxh/git branch 新建一个分支/查看分支
git checkout git checkout wxh /git checkout -d wxh 切换分支/删除分支
git merge git merge wxh 合并某分支到当前分支 

git branch #浏览分支
git branch wxh #创建一个wxh分支
git checkout wxh #
git switch wxh   #均表示切换到wxh分支
git checkout -d wxh #删除wxh分支

上述命令可以简写为

git checkout -b # 或
git switch -c
